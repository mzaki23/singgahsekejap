5 Langkah Deploy Step-by-Step

5.1 Push ke GitHub

1

Inisialisasi Git repository

git init
git add .
git commit -m "initial commit: Explore Batam app"
2

Buat repository di GitHub dan push

git remote add origin https://github.com/USERNAME/explore-batam.git
git branch -M main
git push -u origin main
⚠️
Pastikan file sensitif tidak ikut ter-commit File .env, .env.local, dan node_modules/ sudah ada di .gitignore. Jangan pernah commit nilai API key atau password asli.

5.2 Import ke Vercel

3

Buka Vercel dan import project

Masuk ke vercel.com → klik Add New... → Project → pilih repository explore-batam dari GitHub.

4

Konfigurasi build settings

Vercel akan auto-detect Next.js. Pastikan:

  • Framework: Next.js
  • Build Command: next build
  • Output Directory: .next
  • Node.js Version: 18.x atau 20.x
5

Isi Environment Variables

Sebelum klik Deploy, isi semua variabel di bagian Environment Variables. Lihat daftar lengkap di Bagian 6.

6

Klik Deploy

Vercel akan mulai build. Proses ini memakan waktu 1–3 menit. Pantau log di tab Deployments.

5.3 Update NEXTAUTH_URL

Setelah deploy pertama selesai dan mendapatkan URL Vercel (misal: explore-batam.vercel.app):

7

Update NEXTAUTH_URL di Vercel Dashboard

Pergi ke Settings → Environment Variables, update nilai NEXTAUTH_URL dan NEXT_PUBLIC_APP_URL dengan URL asli yang diberikan Vercel, lalu Redeploy.

6 Environment Variables Lengkap

Isi semua variabel berikut di Vercel Dashboard → Settings → Environment Variables. Pilih environment: Production, Preview, dan Development.

6.1 Database

VariabelNilai / FormatKeterangan
POSTGRES_URL postgres://user:pass@host/db?sslmode=require Koneksi utama (pooled)
POSTGRES_PRISMA_URL ...?pgbouncer=true&connect_timeout=15 Dengan pgbouncer untuk runtime
POSTGRES_URL_NON_POOLING postgres://user:pass@host/db?sslmode=require Tanpa pooling, untuk migrations

6.2 NextAuth

VariabelContoh NilaiKeterangan
NEXTAUTH_URL https://explore-batam.vercel.app URL produksi lengkap
NEXTAUTH_SECRET xK2mP9...<32 byte base64> Generate: openssl rand -base64 32

6.3 Cloudinary

VariabelDariKeterangan

CLOUDINARY_API_SECRET Settings → API Keys API secret (jaga kerahasiaannya!)
CLOUDINARY_UPLOAD_PRESET Settings → Upload Presets Nama preset upload yang dibuat

6.4 App URL

VariabelContoh NilaiKeterangan

🔒
Keamanan: Variabel Sensitif CLOUDINARY_API_SECRET, NEXTAUTH_SECRET, dan semua POSTGRES_* URL bersifat rahasia. Jangan pernah expose ke client-side atau commit ke repository. Variabel yang diawali NEXT_PUBLIC_ akan terekspos ke browser — pastikan isinya aman untuk dipublikasi.

7 Setup Database Production

7.1 Buat Database di Neon

1

Buka neon.tech dan buat project baru

Pilih region terdekat (Singapore jika tersedia) dan beri nama database, misal: explore-batam-prod.

2

Salin connection strings

Di Neon Dashboard → Connection Details, salin tiga format URL (pooled, Prisma, non-pooling) ke Vercel Environment Variables.

7.2 Jalankan Schema SQL

3

Buka SQL Editor di Neon

Neon Dashboard → SQL Editor → New Query.

4

Jalankan file schema.sql

Copy seluruh isi file database/schema.sql dari repository, paste ke SQL Editor, lalu klik Run.

7.3 Verifikasi Tabel

Jalankan query ini untuk memastikan semua tabel berhasil dibuat:

SELECT table_name, column_count
FROM (
  SELECT
    t.table_name,
    COUNT(c.column_name) AS column_count
  FROM information_schema.tables t
  JOIN information_schema.columns c
    ON t.table_name = c.table_name
  WHERE t.table_schema = 'public'
    AND t.table_type = 'BASE TABLE'
  GROUP BY t.table_name
) ORDER BY table_name;

Tabel yang diharapkan muncul: users, places, reviews (sesuai schema.sql)

7.4 Buat Admin User

-- Hash password menggunakan bcrypt cost 10
-- Generate hash di: https://bcrypt-generator.com/ (rounds: 10)
INSERT INTO users (name, email, password_hash, role, status)
VALUES (
  'Admin',
  'admin@explorebatam.com',
  '$2a$10$GANTI_DENGAN_HASH_BCRYPT_ASLI',
  'super_user',
  'active'
);
⚠️
Jangan gunakan plain-text password Selalu gunakan bcrypt hash. Ganti $2a$10$GANTI_DENGAN_HASH_BCRYPT_ASLI dengan hash yang di-generate dari password admin kamu.

9 Troubleshooting

Error Umum dan Solusi

ErrorPenyebabSolusi
500 Internal Server Error Env vars tidak lengkap atau salah format Cek Vercel Logs → Functions, pastikan semua POSTGRES_* terisi
Build failed: Module not found Dependency hilang Jalankan npm install lokal, pastikan package.json lengkap
NEXTAUTH_URL mismatch URL tidak sesuai domain Vercel Update NEXTAUTH_URL di Vercel → Redeploy
Gambar tidak muncul Domain Cloudinary belum di-whitelist Tambahkan res.cloudinary.com ke next.config.js images.remotePatterns
Peta tidak tampil Leaflet SSR error Pastikan komponen Map menggunakan dynamic import dengan ssr: false
Database connection error SSL atau pooling issue Pastikan POSTGRES_URL mengandung ?sslmode=require
Upload gambar gagal Cloudinary preset salah atau API secret keliru Verifikasi CLOUDINARY_UPLOAD_PRESET adalah unsigned preset

Cara Cek Logs di Vercel

!

Vercel Dashboard → Project → Deployments → [Latest] → Functions

Semua error server-side (API routes, SSR) akan muncul di sini secara real-time.

10 Checklist Final

Gunakan checklist ini sebelum mengumumkan aplikasi live:

Persiapan

  • Source code sudah di-push ke GitHub repository
  • File .env dan node_modules tidak ikut ter-commit (ada di .gitignore)
  • Build lokal berhasil: npm run build tanpa error
  • Akun Neon / Vercel Postgres sudah dibuat
  • Akun Cloudinary sudah siap dan upload preset sudah dibuat
  • NEXTAUTH_SECRET sudah di-generate (32 byte base64)

Deploy

  • Project sudah di-import ke Vercel dari GitHub
  • Semua 11 environment variables sudah diisi
  • Build Vercel berhasil tanpa error
  • NEXTAUTH_URL sudah diupdate dengan URL Vercel asli
  • Redeploy dilakukan setelah update NEXTAUTH_URL

Database

  • Schema SQL sudah dijalankan di production database
  • Semua tabel berhasil dibuat dan terverifikasi
  • Admin user sudah dibuat dengan bcrypt hash
  • Koneksi database berhasil dari aplikasi production

Verifikasi Fungsional

  • Homepage load dengan benar dan peta Leaflet tampil
  • Login admin berhasil di /admin/login
  • Akses /admin tanpa login di-redirect ke halaman login
  • Upload gambar berhasil disimpan di Cloudinary
  • CRUD destinasi berfungsi (tambah, edit, hapus)
  • API endpoint /api/places mengembalikan data JSON yang benar
